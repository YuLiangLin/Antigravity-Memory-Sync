# Context Handover（對話交接）

當使用者準備切換模型或結束對話時，執行以下步驟：

// turbo-all

## 步驟

1. **產生交接文件** — 建立或更新 `walkthrough.md` artifact，包含以下內容：

```markdown
# [對話主題]

## 背景
- 使用者的原始需求是什麼

## 已完成的工作
- 列出所有已完成的變更（包含具體檔案路徑）
- 每個變更的簡短說明

## 目前狀態
- 哪些功能已經可以用
- 最新的版本號

## 未完成 / 待辦
- 列出還沒做完的項目
- 列出使用者提到但還沒開始的需求

## 重要決策記錄
- 記錄做過的重要技術選擇和原因

## 相關檔案
- 列出這次修改過的所有檔案（絕對路徑）
```

1. **更新 task.md** — 確保 task.md 反映最新進度（✅ 已完成 / 🔄 進行中 / 📋 待做）

2. **更新 AGENTS.md**（如果有重大變更） — 將新增的功能、版本號等更新到專案的 AGENTS.md

3. **通知使用者** — 告知交接文件已產生，可以安全切換模型
